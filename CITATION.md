# How to cite

This registry is licensed [CC BY 4.0](LICENSE). You're free to reuse, quote, and adapt the data — attribution required.

## Human citation

> *UAE Cybersecurity & Data Protection Regulations Registry* (v3.1, 19 April 2026). Published by nshield.io. Available at: https://github.com/nshield-security/uae-regulations-registry. License: CC BY 4.0.

## Citing a specific mandate

> *UAE Regulations Registry — [Mandate name]* (v3.1). nshield.io. github.com/nshield-security/uae-regulations-registry/blob/main/mandates/master-registry.md

Example:

> *UAE Regulations Registry — CBUAE DL 6/2025 reconciliation deadline* (v3.1, April 2026). nshield.io.

## Machine citation

A `CITATION.cff` file is provided in the repo root. GitHub auto-detects it and surfaces a "Cite this repository" button on the repo page. Tooling that reads CFF (Zenodo, academic reference managers, ingestion pipelines) will pick up the citation automatically.

## Attribution minimum

At minimum:

- Name the registry: *UAE Cybersecurity & Data Protection Regulations Registry*
- Credit the publisher: **nshield.io**
- Link to the repo: `github.com/nshield-security/uae-regulations-registry`
- Note the version and validation date

If you're citing a quickly-changing figure (a deadline within 90 days of its publication, a penalty range within an open rulemaking), check the source against the current primary source before publishing — regulations move.
